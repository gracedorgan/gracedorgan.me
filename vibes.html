Vibes App

Location-based playlists

Background

Vibes is a mobile application that allows users to create collaborative playlists with music from Spotify. However, a user is only allowed to add to the playlist when they are in the physical vicinity of the event. The front end was built with React Native, and the backend was built with MongoDB with a little help from th Spotify API. See the frontend repo here and the backend here. This project was created as a final project for my class CS52, in which the only instructions were to build something cool with what we had learned.

My Contribution

As this was a group project, and we had a small amount of time to finish everything, we split up the workload amongst team members. I worked exclusively on the backend, using axios to make Spotify API calls, and Mongo to query our database of user-created playlists.

What Went Well

I was lucky to work with an amazing team of women on this project who were able to mentor me throught the whole process of design and development. I found Mongo to be easy to scale up from the "practice projects" I had done earlier to this larger and more complex project.

What Could Have Gone Better

When I began this project, I still did not feel entirely comfortable with the React framework, especially regarding how data is passed around. Thanks to this, I spent longer than I would have liked figuring out how to structure the varous controllers, and how to call each method.

What I Learned

I learned it is possible to effectively delegate responsibility for different components of a project if there is strong enough communication. I also solidified my understanding of React and MongoDB.
