Tiny Search Engine

Searching for the little things

Background

This search engine is based on the paper "Searching the Web" by Arvinda,Cho,Hector,Paepcke,Rsram. As such, it has three main parts: a crawler, an indexer, and a querier. I wrote each program in C, using custom, built-from-scratch Sets, Bags, and Hashtables. The engine was tested thoroughly with the help of BASH scripts and a "closed loop" made by the Dartmouth computer science department.

My Contribution

This project was for a class, so I wrote all of the code in the project.

What Went Well

I grasped how each part of the engine should come together, and did not have much trouble linking all of the parts together. Over all the program turned out to be robust, and passed all testing cases given to it.

What Could Have Gone Better

I struggled a lot with memory leaks. Coming from a lot of previous experience in Java, dealing with low-level memory allocation wasn't something I used to dealing with, so I spent a lot of time freeing up memory.

What I Learned

I learned C in-depth to write this code. I also streamlined my debugging process to more quickly identify issues.
